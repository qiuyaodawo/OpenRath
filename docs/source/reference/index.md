# Python API reference

This page mirrors the organization of the [PyTorch reference index](https://docs.pytorch.org/docs/stable/pytorch-api.html):
short descriptions per package; detailed autogenerated signatures will arrive once
`sphinx-autosummary` is wired to `src/rath`.

## Top-level package

* **`rath`** — Lazy exports for `backend`, `flow`, and `session` (session loads on first attribute access).

## Session runtime

* **`rath.session`** — `Session`, chunk utilities, session loop (`run_session_loop`, `SessionLoopExecutor`),
  primitives (`fork_session`, …), lineage graph helpers, registry (`session_registry`).
* **`rath.session.chunk`** — `ChunkTable`, row helpers, conversion to chat messages.
* **`rath.session.graph`** — `SessionLineage`, `LineageRecorder`, traversal utilities.
* **`rath.session.loop`** — Loop implementation shared by workflows.
* **`rath.session.manager`** — `SessionRegistry`, global registration hooks.
* **`rath.session.provider_builtin`** — `DefaultSessionLoopExecutor`.
* **`rath.session.primitives`** — Experimental session transforms.

## Flow orchestration

* **`rath.flow`** — Re-exports `Workflow`, `Provider`, `Agent` from submodules.
* **`rath.flow.workflow`** — `Workflow`, `run_session_loop_from_agent`.
* **`rath.flow.agent`** — `Agent` dataclass bundling system session + provider.
* **`rath.flow.tool`** — `FlowToolCall`, factories, `ToolTable`, global registration helpers.

## Backends

* **`rath.backend`** — Public facade: registry functions, sandbox types, results, streams, capability enums.
* **`rath.backend.abc`** — `Backend`, `BackendSandbox`, `BackendSandboxSpec` protocols.
* **`rath.backend.local`** — Host subprocess backend.
* **`rath.backend.opensandbox`** — Remote sandbox adapter (optional extra).
* **`rath.backend.registry`** — Name → backend class registration.
* **`rath.backend.results`** — `CommandResult`, `FileContent`, `CodeResult`, failure structs.
* **`rath.backend.stream`** — `Stream`, `Event`, `Future`.
* **`rath.backend.tool_types`** — Canonical backend-facing tool dataclasses (aliases for flow tools).

## LLM integration

* **`rath.llm`** — Package exports for client, provider, chat request/response types.
* **`rath.llm.client`** — `RathOpenAIChatClient`.
* **`rath.llm.provider`** — `Provider` sampling container.
* **`rath.llm.chat_request`** / **`rath.llm.chat_response`** — Wire-aligned dataclasses.
* **`rath.llm.openai_create_kwargs`** / **`openai_normalize`** — Translation helpers.

## Utilities

* **`rath.utils.env`** — Environment helpers shared by LLM settings loaders.
